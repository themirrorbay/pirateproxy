The technical side of The Pirate Bay

The website had Hypercube webserver as an old and deprecated technology a long time ago. Since then, pirate bay developers upgraded it by making it less hardware-heavy, reducing bandwidth, and making it easier to understand.

At this time, the server was getting two requests per millisecond on each of four Linux shared intermediate servers, so the team moved to more than 30 different servers to help ease traffic which helped the internet speed.

The pirate bay website runs eight web parts, one global database, multiple search systems, and numerous BitTorrent files trackers. The frontend is written on PHP using Lighttpd, and the previously mentioned database is MySQL. Search systems are powered with Sphinx, and everything on the website gets cached by Varnish.

In 2007 backend finally migrated from Hypercube to an open-source solution called Opentracker. Also, SSL encryption was introduced around that time to prevent ISPS (internet service providers) from seeing detailed information about requests and to secure its user base. This migration made it possible to synchronize dozens of servers using UDP multicast.

IPv6 support was turned off entirely from Opentracker in 2009 since many people at that time were already introduced to decentralized ways of torrenting with things like Distributed Hash Table (DHT) and Peer Exchange (PEX).

At the end of 2012, magnet links were announced as the only way of getting torrent information from the pirate bay website since it would be beneficial for running costs, the reason behind it is additionally storing torrent files would drive server prices higher than it could be with Magnet Link technology.
